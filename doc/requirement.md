## 一、問題陳述

台灣現有事實查核機構（TFC）的結構性問題：
- 4-5 名全職記者，人均月產 1.7 篇，82% 判「錯誤」= 只打軟目標
- 選題偏差 2.2 倍（保護綠營 vs 藍營），verdict 寬嚴差異 2.2 倍
- 系統性迴避灰色地帶（中共認知作戰結構、美國對台承諾、促統論述）
- 壟斷 IFCN 認證 + Facebook/LINE 合作 = 單一守門人無外部監督
- 查核流程本質是 Google 反搜 + 打電話，可大幅自動化

（以上結論來自 tfc-audit 專案，BERTopic 全量分析 4,125 篇報告）

## 二、既有開源專案參考

| 專案 | 架構 | 優點 | 缺點 |
|------|------|------|------|
| **Loki** (Libr-AI, 1.1k⭐) | 五步 pipeline: decompose → check-worthiness → query → evidence → verify | 最完整的端到端系統，支援多模態（文字/語音/圖片/影片） | 英文only，依賴 OpenAI + Serper API，無社群協作機制 |
| **ClaimeAI** (LangGraph, 98⭐) | Claimify 拆解 → Tavily 搜尋 → 投票消歧 → 報告 | LangGraph 工作流清晰，有 Web UI | 英文only，純 LLM 判定無人工介入，無中文支援 |
| **OpenFactCheck** (59⭐) | 統一框架：Response Evaluator + LLM Evaluator + Fact Checker Evaluator | 可插拔不同查核引擎，有 benchmark 評估 | 偏學術評估，非生產系統 |
| **Factcheck-GPT** | 查核 LLM 輸出的事實性 | 有標註 + 評估雙模式 | 只查 LLM 輸出，不查社群謠言 |
| **ProgramFC** (ACL 2023) | 程式導向推理查核複雜聲明 | 可處理多步推理 | 研究原型，非生產系統 |

### 共同缺口（= 我們的機會）

1. **全部英文**，無中文社群謠言處理能力
2. **全自動判定**，無人工介入機制 → LLM 幻覺風險
3. **無社群協作**，無公民記者認養機制
4. **無審計功能**，無法分析查核機構自身的偏差
5. **無本地推論**，全依賴雲端 API → 成本高、隱私差

## 三、系統定位

**不是取代 TFC，是讓 TFC 這種機構變得不必要。**

核心差異：

| | TFC 模式 | verifact 模式 |
|---|---|---|
| 選題 | 編輯室 4-5 人決定 | 公民自選認養 → 市場機制修正偏差 |
| 查證 | 記者手動反搜+打電話 | Agent 自動備料 + 人工最後一哩 |
| 判定 | 蓋「錯誤」章 | 呈現證據鏈，讀者自行判斷 |
| 速度 | 數天 | 分鐘級（自動部分）+ 認養後數小時 |
| 透明度 | 報告成品 | 全流程公開（證據來源、信心分數、推理過程） |
| 規模 | 22 人 | 無上限公民記者 |
| 監督 | 無外部監督 | 開源 + 數據公開 = 任何人可審計 |

## 四、架構

```
┌─────────────────────────────────────────────────┐
│                   verifact                       │
├──────────┬──────────┬──────────┬────────────────┤
│  Intake  │  Triage  │  Agent   │  Adopt & Pub   │
│  爬蟲層   │  分類層   │  查證層   │  認養與發布層   │
└──────────┴──────────┴──────────┴────────────────┘
```

### Layer 1: Intake（爬蟲層）

- Playwright 定時爬取：LINE 群組熱傳、FB 公開社團、PTT、X、TikTok
- 去重：embedding 相似度 > 0.9 合併為同一則
- 熱度偵測：短時間內多平台出現 → 優先處理

### Layer 2: Triage（分類層）

- BERTopic 聚類：自動歸類到已知主題
- 類型標籤：圖片造假 / 影片變造 / 引述捏造 / 數據錯誤 / 脈絡誤導 / 觀點偽裝成事實
- 難度評估：Easy（反搜可解）/ Medium（需查公開資料）/ Hard（需人工查證）
- 重複偵測：與已查核資料庫比對（含 TFC/MyGoPen/Cofacts 歷史）

### Layer 3: Agent（查證層）

參考 Loki 五步 pipeline + ClaimeAI 的 Claimify 拆解，但改為中文 + 本地推論：

```
3a. Decompose（拆解聲明）
    - LLM 拆解為可查核的原子聲明
    - 處理代名詞、省略、隱含前提
    
3b. Evidence Retrieval（證據檢索）
    - 反搜圖：Google Lens API / TinEye
    - 公開資料：政府公報、立院議事錄、主計處、CDC、NAER
    - 學術文獻：PubMed、Google Scholar
    - 既有查核：TFC/MyGoPen/Cofacts API
    - Web search：Serper / Tavily / SearXNG（自架）
    
3c. Verify（初步驗證）
    - 本地 LLM（Hermes/OpenClaw/Gemma）比對證據與聲明
    - 輸出：每條聲明的 evidence chain + confidence score
    - 不輸出「對/錯」，輸出「支持/反駁/證據不足/需人工」
    
3d. Package（打包題目）
    - 生成「題目包」：原始謠言 + 拆解聲明 + 已找到證據 + 缺什麼 + 建議查證方向
    - 標記難度和預估工時
```

### Layer 4: Adopt & Publish（認養與發布層）

- 公民記者瀏覽題目包，認養感興趣的
- 補上最後一哩：打電話、實地查證、專家訪談
- 提交查核報告（含證據鏈）
- 同儕審核（至少一人 review）
- 發布：網站 + RSS + LINE Bot + API

## 4.5、謠言偵測：語言指紋 + 類目分類

BERTopic 是分析工具，不是判斷工具。判斷「值得查核」靠三層：

### 謠言的語言指紋（從 TFC 4,125 篇提取）

| 特徵 | 覆蓋率 | 說明 |
|------|--------|------|
| 標題含「網傳」 | **87%** | 幾乎是謠言的身分證 |
| 標題含「引述」（「…」） | **87%** | 謠言愛引述不存在的話 |
| 標題以「？」結尾 | **81%** | 疑問句式是謠言標配 |
| 摘要以「網傳」開頭 | 1,921 句 | |
| 摘要含「傳言稱/說/指」 | 828 句 | |
| 摘要含「影片稱/說」 | 563 句 | |
| 情緒詞（千萬/注意/小心/緊急/竟然） | 標題出現 200+ 次 | |

→ 用 regex 就能抓到 80% 的謠言格式。

### TFC 官方類目（= 謠言的偏好領域）

| 類目 | 佔比 | 說明 |
|------|------|------|
| 國際 | 29.8% | 最大宗：假國際新聞、變造影片 |
| 生活 | 20.6% | 食安、偏方、都市傳說 |
| 政治與政策 | 19.2% | 選舉、政策、政治人物 |
| 健康 | 19.1% | 疫苗、藥物、養生 |
| 科技資安 | 4.4% | 詐騙、個資、APP |
| 環境能源 | 3.0% | 核電、空污、氣候 |
| 科學研究 | 1.3% | |

→ 四大類（國際+生活+政治+健康）佔 **88.7%**。

### 三層偵測機制

```
新內容進來
  ├─ Layer A: 語言指紋（regex）
  │   「網傳」+「？」+ 引述 → 命中 → 直接進 triage
  │
  ├─ Layer B: 語意比對（embedding kNN）
  │   與 TFC/Cofacts 歷史 4,125+ 篇比對
  │   相似度 > 閾值 → 進 triage（可能是舊謠言翻新）
  │
  ├─ Layer C: LLM check-worthiness
  │   「這段文字是否包含可查核的事實聲明？」
  │   分數 > 閾值 → 進 triage
  │
  └─ 都沒命中 → 觀察池，等熱度上升再處理
```

本質上就是：**謠言有固定的語言模板和偏好類目，用已知的標記資料做分類器，不需要從零判斷。**

## 八、關鍵設計原則

1. **不蓋章**：不輸出「錯誤/正確」，輸出證據鏈讓讀者判斷
2. **全透明**：每一步的推理過程、證據來源、信心分數全部公開
3. **可審計**：平台自身的選題分布、認養模式、查核結果全部可分析
4. **本地優先**：核心 ML 推論全部本地跑，不依賴雲端 API
5. **開源**：MIT License，任何人可 fork、可驗證、可改進
6. **低門檻**：公民記者不需要技術背景，題目包已備好 80% 的料
7. **管線化供應**：pipeline 產出的題目池、證據包、API 全部開放，歡迎任何查核機構（含 TFC）選題、引用、串接

### 第 7 點的戰略意義

verifact 的定位不是 TFC 的競爭者，是**事實查核的公共基礎設施**：

```
verifact pipeline（自動產出）
    │
    ├→ 公民記者認養
    ├→ TFC 選題引用（歡迎，MIT）
    ├→ MyGoPen 選題引用
    ├→ Cofacts 社群引用
    ├→ 新聞系學生作業
    └→ 任何人
```

- 題目池公開 = 選題偏差自動透明化（誰挑了什麼、不挑什麼，一目了然）
- 證據包預備好 = 查核機構的工作從「找證據」降級為「寫報告」
- MIT 授權 = 無法拒絕、無法反對、無法說你在搶生意
- API 開放 = LINE Bot、Facebook 外掛、瀏覽器擴充都能串接

**最終效果：不是取代守門人，是讓守門這件事變成公共財。**

## 九、風險與對策

| 風險 | 對策 |
|------|------|
| LLM 幻覺生成假證據 | Agent 只備料不判定，人做最終判斷；證據必須附原始連結 |
| 公民記者品質參差 | 同儕審核 + 信譽系統 + 新手只能認養 Easy 題目 |
| 被告誹謗 | 不蓋「錯誤」章，只呈現證據鏈；法律免責聲明 |
| 被政治勢力滲透 | 選題由市場機制決定，非編輯室；數據公開可審計 |
| 冷啟動無人認養 | Phase 1 先用 W10 學生做種子用戶；與 Cofacts 社群合作 |
| 爬蟲被封鎖 | 多帳號輪替 + SearXNG 自架 + 接受舉報作為補充 |
| **LLM 灌水攻擊** | 見下方反灌水機制 |

### 反灌水機制

平台歡迎所有人提交證據，但必須可追溯、可驗證：

**規則：每條證據必須附來源連結。沒有連結 = 意見，不是證據。**

其餘交給社群自己處理：

1. **連結驗證**：AI 自動檢查連結是否存在、內容是否與聲明相關。死連結或內容不符自動標記。
2. **AI 生成標記**：疑似 LLM 生成的提交公開標記「🤖 疑似 AI 生成」。不刪除，讓所有人看到。
3. **社群核實**：任何人都可以去核對別人提交的文字與連結。發現連結內容與聲明不符 → 標記 → 該提交降權。
4. **信譽加權**：過去提交的證據被社群驗證通過越多，你的票越重。靠灌水起家的帳號權重自然低。

**核心邏輯：不替你判斷真假，但讓造假的成本高於說真話。**
